# SAM2

## Overview

SAM 2 是面向图像和视频分割的统一模型。  
在图像模式下, 它和 SAM 类似;  
在视频模式下, 它通过记忆机制处理时序信息。

其核心思想是:

* 每帧先编码成视觉特征
* 当前帧特征和历史记忆交互
* 用户提示决定关注对象
* 掩码解码器输出当前分割结果

---

## 核心组件

### 1. Image Encoder

图像编码器对每一帧提取视觉特征:

$$
F_t=E(I_t)
$$

在交互过程中, 每帧通常只需要编码一次。

### 2. Memory Attention

这是 SAM 2 处理视频的关键模块。  
它将:

* 当前帧特征
* 历史记忆
* 当前提示

统一到一个注意力框架中融合。

### 3. Prompt Encoder

提示编码器负责处理:

* 正负点击
* 边界框
* 掩码提示

因此模型可以在交互式设置下逐步细化结果。

### 4. Mask Decoder

掩码解码器根据当前帧特征和提示生成分割结果。

相较于静态图像分割, 视频模式里还会额外考虑:

* 遮挡判断
* 同类物体歧义
* 当前帧最优掩码传播

---

## 记忆机制

SAM 2 的视频能力依赖于记忆系统:

1. 把先前帧的掩码与图像特征编码成 memory
2. 维护一个 memory bank
3. 当前帧通过 cross-attention 查询这部分历史记忆

常见存储内容包括:

* 最近若干帧的 memory
* 用户提示过的关键帧 memory
* object pointers

因此它可以在视频中维持对象的一致性。

---

## 总结

SAM 2 可以看成是在 SAM 的图像分割框架上, 增加了一套流式视频记忆机制。  
核心不只是“分当前帧”, 而是“利用历史信息连续地分对象”。
